This playbook is organized into six thematic sections that follow the natural progression of an AI-Enabled Problem Solving engagement. Each section builds on the previous, but can also be used as a standalone reference for practitioners at any stage of their engagement.

<ProcessFlow steps={[
  { title: "Diagnosis & Prioritization", description: "Problem framing, data ingestion, AI-assisted analysis, hypothesis testing, and opportunity synthesis." },
  { title: "Opportunity Prioritization", description: "Documentation, WSJF scoring, Value vs. Complexity matrix, roadmap sequencing." },
  { title: "Build & Deliver", description: "Solution design, prototyping, Agile delivery, SAFe for multi-team coordination." },
  { title: "Sustain & Scale", description: "Three pillars framework, performance monitoring, drift prevention, agile governance." },
  { title: "Change Management", description: "Readiness baseline, influence network mapping, communications planning, governance design." }
]} />

<Callout type="tip" title="How to Use This Playbook">
Each section builds on the previous, but the methodology is inherently non-linear. You may find yourself revisiting earlier sections as new data and insights emerge. The playbook is designed to support this iterative approach.
</Callout>
