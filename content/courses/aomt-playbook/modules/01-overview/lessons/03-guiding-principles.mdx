The AI-Enabled Problem Solving methodology is grounded in principles that distinguish it from traditional consulting approaches. These principles guide every decision in the diagnostic and solution development process.

<ProcessFlow steps={[
  { title: "Data First", description: "Always start with data before forming conclusions. Ingest broadly, analyze deeply, and let patterns emerge from evidence rather than assumptions. Do not begin hypothesis generation before reviewing available data." },
  { title: "AI First", description: "Design solutions that leverage the full capabilities of AI. Ask \"what becomes possible if AI handles this?\" rather than \"how can AI speed up one step?\" This approach often yields fundamentally different and more effective solutions." },
  { title: "Validate Everything", description: "Never rely on a single AI analysis for critical decisions. Use multiple runs, prompt variations, cross-tool comparison, and qualitative confirmation from domain experts." },
  { title: "Iterative Process", description: "Diagnosis is non-linear. Revisit problem statements, hypotheses, and analysis plans as new data emerges. Iteration is the design, not a flaw — each cycle sharpens the findings." },
  { title: "Curiosity-Driven", description: "The talent for asking the right questions is the key to successful AI-Enabled Problem Solving. Maintain an exploratory mindset — unexpected findings often reveal the most important insights." }
]} />
